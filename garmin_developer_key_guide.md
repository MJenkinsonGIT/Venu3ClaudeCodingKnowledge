## 1. What Is a Developer Key?

A **developer key** is a cryptographic private key used to **digitally sign** Connect IQ apps. Signing is required for:
- Installing apps on physical Garmin devices
- Publishing apps to the Connect IQ Store
- Ensuring app authenticity and integrity

**Simulator testing does NOT require a developer key.**

---

## 2. Key Requirements

### Format
- **DER format** (Distinguished Encoding Rules) — binary encoding
- **PEM format will NOT work** (text/Base64 encoding)
- The compiler specifically needs PKCS#8 DER format

### Algorithm
- RSA private key
- 4096-bit key size (recommended)

### Location
- Common name: `developer_key.der`
- Store at: `C:\Users\[YourName]\AppData\Roaming\Garmin\ConnectIQ\`

## 3. How to Generate a Developer Key

### Prerequisites
- OpenSSL installed (download from https://slproweb.com/products/Win32OpenSSL.html — choose "Win64 OpenSSL v3.x.x Light", or use Git for Windows which includes OpenSSL)

### Official Two-Step Method (Only Reliable Method)

**Step 1: Generate PEM key**
```powershell
cd C:\Users\[YourName]\AppData\Roaming\Garmin\ConnectIQ

& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" genrsa -out developer_key.pem 4096
```

**Step 2: Convert to DER (what Garmin actually needs)**
```powershell
& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" pkcs8 -topk8 -inform PEM -outform DER -in developer_key.pem -out developer_key.der -nocrypt
```

**Notes:**
- `-nocrypt` = no password protection on the key
- The PEM file is intermediate only — the `.der` file is what VS Code uses
- You can delete `developer_key.pem` after conversion

**Single-command alternative (not recommended):** Some OpenSSL versions can generate DER directly but this is unreliable. Always use the two-step method.

## 4. Configuring VS Code

### Method 1: User Settings (Recommended)

1. Press **Ctrl+Shift+P**
2. Type: `Preferences: Open User Settings (JSON)`
3. Add these lines:

```json
{
    "monkeyC.sdk": "C:\\Users\\[YourName]\\AppData\\Roaming\\Garmin\\ConnectIQ\\Sdks\\connectiq-sdk-win-8.4.1-2026-02-03-e9f77eeaa",
    "monkeyC.developerKeyPath": "C:\\Users\\[YourName]\\AppData\\Roaming\\Garmin\\ConnectIQ\\developer_key.der"
}
```

**Critical:** Use **double backslashes** (`\\`) in JSON paths on Windows.

4. Save (Ctrl+S)
5. Reload VS Code: Ctrl+Shift+P → `Developer: Reload Window`

### Method 2: Command Line

```bash
monkeyc -d venu3 -f monkey.jungle -o output.prg -y C:\path\to\developer_key.der
```

## 5. Common Errors and Solutions

| Error | Cause | Solution |
|-------|-------|----------|
| "Unable to decode key" | Wrong format (PEM instead of DER) | Redo two-step generation |
| "Unable to load private key: unknown version: 3" | Wrong algorithm or incompatible format | Delete key, use official two-step method with PKCS#8 second step |
| "Unable to load private key: undefined" | VS Code doesn't know key path | Set `monkeyC.developerKeyPath` in settings |
| "Key file not found" | Wrong path or filename | Verify path with double backslashes; confirm `.der` extension |

## 6. What the Key Looks Like

### PEM Format (Intermediate)
```
-----BEGIN RSA PRIVATE KEY-----
MIIJKAIBAAKCAgEA...
[many lines of Base64]
...GnQ1aNZpw==
-----END RSA PRIVATE KEY-----
```

### DER Format (Final)
When opened in a text editor: binary/unreadable characters, possibly starting with `0` or other symbols. **This is correct.** DER is binary, not human-readable. Do not edit this file.

## 7. Security Considerations

- **Never** commit your developer key to Git or any public repository
- **Never** share your key
- Add `developer_key.*` to `.gitignore`
- Keep a secure backup — apps signed with old key won't match new key if regenerated
- If compromised: generate new key, update VS Code settings, rebuild all apps

---

## 8. Testing Without a Key

You can test in the simulator without a developer key:
1. Remove or comment out `developerKeyPath` in VS Code settings
2. Build for simulator only (F5)

You cannot deploy to physical devices without signing.

---

## 9. Troubleshooting Checklist

- [ ] OpenSSL is installed
- [ ] Key generated using the **two-step method** (not single command)
- [ ] Key file is named `developer_key.der` (not `.pem`)
- [ ] Key is at: `C:\Users\[YourName]\AppData\Roaming\Garmin\ConnectIQ\`
- [ ] VS Code settings path uses **double backslashes**
- [ ] VS Code reloaded after changing settings
- [ ] Key file not corrupted (regenerate if unsure)

```powershell
& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" genrsa -out developer_key.pem 4096
& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" pkcs8 -topk8 -inform PEM -outform DER -in developer_key.pem -out developer_key.der -nocrypt
```

### VS Code settings.json
```json
{
    "monkeyC.developerKeyPath": "C:\\Users\\[YourName]\\AppData\\Roaming\\Garmin\\ConnectIQ\\developer_key.der"
}
```

### Verify
Build any project — if it compiles without "Unable to load private key" errors, the key is working.
